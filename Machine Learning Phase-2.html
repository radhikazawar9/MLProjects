Apartment Price Investigation - Phase II¶

The objective of this phase is to fit and compare three different regressors to predict the price of apartment. This is a follow-up part to the phase I where We have understood, processed, prepared data for the machine learning algorithm. The data has 19 descriptive features and one target feature. The target feature, SoldPrice, is numeric and the descriptive features include 12 numeric feature and 6 categorical features.

  • Section 2. Overview - methodology outline
  • Section 3. Data Preparation - brief dataset summary and method evaluation styrategy

  • Section 7. Summary - provides a brief summary of our work in this project

Overview¶

Methodology:¶

Following regression algorithms are used to predict the target feature.

  • Linear Regressor
  • K-Nearest Neighbors Regressor
  • Decision trees Regressor

The modeling strategy begins by transforming the dataset which was cleaned in Phase I. This transformation includes encoding categorical descriptive features as numerical and then scaling of the descriptive features. Then the data was split on 70%-30% split. At the end of splitting, training set has 4123 and test set has 1768 rows.

Before fitting a perticular model, we have selected importatnt feature using various feature selection methods. In this step, we have used F-score, Mutual Information, Random Forest Importance. Further we also compared the performanve of the different feature selection methods.

Using all the feature together with hyperparameter search, we conduct a 5-fold cross-validation to fine-tune hyperparameters of each regressor using R-square as the performance verifier. We build each model using parallel processing with "-2" cores.

Once the best model is identified for K-Nearest Neighbors Regressor and Decision trees Regressors using a hyperparameter search on the training data, We present visual plot to demonstrate performance of each algorithm. Further, we conduct a 5-fold cross-validation on the test data and perform a paired t-test to see if any performance difference is statistically significant. In addition, we compare the Regressors with respect to their scores on the test data.

Hyperparameter Tuning¶

K-Nearest Neighbors (KNN) :¶

As we have already considered feature selection, we now we stack grid search for KNN hyperparameter tuning via cross-validation.

Performance Comparison¶

We have optimized each one of the three regressor using the training data. We now fit the optimized models on the test data in a cross-validated fashion. But since cross validation itself is a random process, we perform pairwise t-tests to determine if any difference between the performance of any two optimized regressor is statistically significant. First, we perform 5-fold cross-validation on each best model (without any repetitions). Second, we conduct a paired t-test for the r-square score between the following model combinations:

  • Linear regression Vs K-nearest neighbour
  • Decision tree regressor Vs Linear regression
  • K-nearest neighbour Vs Decision tree regressor

Limitations and Proposed Solutions¶

Our modeling strategy has a few flaws and limitation. First, our target variable SoldPrice was continuous in nature and it had big range of data in an uneven distribution. Because of this problem most our algorithm were not able to predict correct values for high price range of apartments as these values were treated as outliers.

Further, For K-nearest neighbor model, the requered computation time was quite high. To avoid this problem either number of observation could be restricted or machine with more computation power should be used.

Lastly, we worked with only a subset of the data. Since data is always valuable, we could re-run our experiments with the entire data while making sure that the train and test split is performed in a proper manner.

Summary¶

The K-nearest neighbour regressor with 3 nearest neighbour for Manhattan distance produces the highest r-square value that is 98.5% which is exceptionally good. The Mean absolute error was also lowest for KNN regressor which is 8611.9. After performing feature selection with linear regression as wrapper, it came to light that the result was optimum on considering all the features which was also confirmed by statistical test. In hyper parameter tuning, we uncover the best parameters for Decision tree regressor and K-nearest neighbour regressor. We also demonstrated performance of all the model by presenting scatter plot of their predicted prices to actual prices respectively along with the residual plot for each algorithm for deeper understanding. After running t-test on the model it comes to light that K-nearest neighbour regressor with n=3 and p=1 outperforms all the other model under investigation.
